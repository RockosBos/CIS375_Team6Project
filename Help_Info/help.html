Help Page

This page is intended to help the user navigate the test case generator

Description

This Test case Generator is for a user looking to create test cases using a series of operations and Relationships between these operations. The original requirement was to create a web application that "Allows the User to enter all operations and their relationships from a behavioral model. The tool then creates a set of test cases that exercise the relationships among operations." We developed the application to meet the following requirements.

  1. User can enter software operations and relationships and the application will generate test cases based on operations correctly calling one another
  2. Create an established help page that allows the user to understand the software.
  3. User can create as many operations and relatiosnships as they want.

Steps

  1. Add/Delete rows to get the desired number of operations.

  2. Type descriptions into each operation name.

  3. Add relationships to other operations using the operation ID. To add multiple, seperate them by spaces.

  4. Click the generate test cases button.

  5. Type PASS/FAIL in the results column to keep track of test case results.

  6. Go back and edit operations OR click the Reset Page button.

Errors

You have inserted an invalid relationship ID, they must be between 0 and the largest ID you have created, A relationship also cannot be the same ID and the Operation ID, see HELP for input instructions.

Potential Fixes

  1. Seperate relationships with spaces.
  2. Ensure that a relationship ID is not equal to the operation ID for a row (Operations cannot be a relationship with themselves)
  3. Make sure there is no leading or ending whitespace in the relationships box.

You have inserted an invalid input, please enter a series of numbers separated by a space. See HELP for input instructions.

Potential Fixes

  1. Relationship Input's should be numbers seperated by spaces. This should be in the format (1 2 3 4) to have a relationship with the first 4 operations.
  2. There should not be non numeric characters in the relationships box. There should only be numbers.

All operations must have a name!

Potential Fixes

  1. Make sure you input a name into the Operation Name field.
